# 小红书监控器 - 快速入门指南

## 5 分钟快速上手

### 第一步：安装依赖 (1 分钟)

```bash
# 进入项目目录
cd /Users/sgl/PycharmProjects/Python_AnalysisWebandSubweb

# 安装依赖
pip3 install -r requirements.txt
```

### 第二步：启动 GUI (1 分钟)

```bash
# 运行启动脚本
python3 run_xhs_monitor_gui.py
```

### 第三步：添加账号 (1 分钟)

1. 在弹出的 GUI 界面中，点击 **"添加账号"**
2. 输入你的小红书账号和密码
3. 点击 **"保存"**

> ⚠️ 首次使用会生成主密码，请妥善保管！

### 第四步：配置搜索 (1 分钟)

1. 在 **"关键词"** 框中输入要搜索的词 (每行一个)
2. 选择 **"时间范围"** (推荐：最近 1 周)
3. 确认 **"提取评论"** 已勾选

### 第五步：开始监控 (1 分钟)

1. 点击 **"开始监控"**
2. 观察日志输出
3. 等待完成提示

### 第六步：查看结果

1. 点击 **"打开结果目录"**
2. 查看生成的 JSON 和 CSV 文件

## 常用操作

### 添加多个账号

```
1. 点击"添加账号"
2. 输入第一个账号 → 保存
3. 继续点击"添加账号"
4. 输入第二个账号 → 保存
5. 重复以上步骤添加更多账号
```

### 使用自动轮换

```
1. 确保勾选"启用自动轮换"
2. 账号选择保持"自动轮换"
3. 程序会自动选择最佳账号
4. 每个账号有 1 小时冷却时间
```

### 导出配置

```
1. 配置好关键词和选项
2. 菜单：文件 → 导出配置
3. 保存为 config.json
4. 下次可直接导入使用
```

### 查看账号状态

```
1. 查看账号列表下方的状态栏
2. 绿色✅ = 正常
3. 黄色⚠️ = 可疑
4. 红色🚫 = 受限
5. 灰色❌ = 封禁
```

## 故障排查

### 问题：点击"开始监控"没反应

**解决**:
1. 检查是否添加了账号
2. 检查关键词是否为空
3. 查看日志区域是否有错误信息

### 问题：自动登录失败

**解决**:
1. 第一次使用需要手动登录一次
2. 登录状态会被保存
3. 后续会自动使用保存的状态

### 问题：账号被限制

**解决**:
1. 暂停使用该账号 24-48 小时
2. 手动登录账号，正常浏览
3. 进行一些真人互动 (点赞、收藏)

### 问题：找不到结果文件

**解决**:
1. 点击"打开结果目录"按钮
2. 或手动查看 `xhs_browser_data` 目录
3. 文件命名格式：`monitor_result_YYYYMMDD_HHMMSS.json`

## 最佳实践

### 单账号使用

```
- 每天最多运行 2-3 小时
- 搜索间隔设置 45-90 分钟
- 不要勾选"无头模式"(观察运行)
- 定期手动登录账号，正常浏览
```

### 多账号使用 (推荐)

```
- 启用"自动轮换"
- 添加 3-5 个账号
- 每个账号每天最多 1 小时
- 账号状态异常时自动跳过
```

### 关键词策略

```
- 每组 3-5 个相关关键词
- 避免过于敏感的词汇
- 定期更换关键词组合
- 使用长尾词更精准
```
